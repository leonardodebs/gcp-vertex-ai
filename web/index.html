2. BigQuery ML — treinar ML com SQL ⭐ único do GCP

A AWS não tem equivalente nativo (o mais próximo é o Redshift ML, que por baixo provisiona o SageMaker). No GCP você treina, avalia e serve um modelo sem sair do SQL: 

CREATE OR REPLACE MODEL alerts_dataset.severity_classifier
OPTIONS (model_type='logistic_reg', input_label_cols=['severity'])
AS SELECT text, severity FROM alerts_dataset.training_data;

SELECT * FROM ML.EVALUATE(MODEL alerts_dataset.severity_classifier);
SELECT * FROM ML.PREDICT(MODEL alerts_dataset.severity_classifier, ...);

6. Quando escolher cada nuvem

Escolha GCP / Vertex AI

  • ML sobre dados que já vivem num warehouse → BigQuery ML
  • Inferência LLM barata e rápida (Gemini Flash)
  • Contexto gigante (1M+) ou multimodal (áudio/vídeo)
  • Equipe forte em SQL/analytics, fraca em MLOps

Continue na AWS

  • Claude é o modelo principal (topo em raciocínio/código)
  • Já há peso de workloads/identidade/rede na AWS
  • Precisa do catálogo mais amplo de serviços gerenciados

Multicloud é viável: a camada de prompts/LLM é portável; o lock-in real está nos dados (BigQuery vs Redshift) e no RAG gerenciado.
